Paste a Cursor mcp.json-style config below. Each server is mounted at localhost:8000/<name>/mcp (raw passthrough), and the aggregate endpoint at localhost:8000/mcp exposes every running server's tools, prompts, and resources under the <server>__<name> namespace (resource URIs keep their original form). The aggregate is the recommended way to wire Cursor — one entry, every backend.

Configuration
Cursor mcp.json (aggregated) Recommended

One Cursor entry, every running backend's tools and prompts. Names are namespaced as <server>__<tool> so they don't collide. Use this unless you have a specific reason to talk to a single backend directly. 

{
  "mcpServers": {
    "zelosmcp-aggregate": {
      "type": "streamable-http",
      "url": "http://localhost:8000/mcp"
    }
  }
}
Cursor full mcp.json

One entry per running backend at /<name>/mcp (raw passthrough, original tool names) plus the aggregate. Use when you need a backend's tools to keep their unprefixed names, or to wire a single backend into a separate Cursor profile. 

{
  "mcpServers": {
    "zelosmcp-aggregate": {
      "type": "streamable-http",
      "url": "http://localhost:8000/mcp"
    }
  }
}
Rules

Comprehensive rule listing every tool from every currently-loaded backend, with descriptions, arg summaries, and a [readonly]/[mutates]/[destructive]/[?] mutability marker. Tool use — Priority (default) tells the agent to prefer MCP tools over shell commands and includes a curated playbook for the mandatory backends; Available emits a neutral catalog with no prioritization. Read-only mode forbids the agent from calling mutating tools — safe default for inspection-style projects (code review, demos). Switch to Read-write when the agent needs to make changes through the MCPs. Save as .cursor/rules/zelosmcp.mdc in any workspace (or ~/.cursor/rules/ for global). Save as .github/copilot-instructions.md (picked up by VS Code Copilot).

Token counts use tiktoken's cl100k_base encoding when available; otherwise a len/4 heuristic.
